\abstract{Testing is one out of many ways to verify the behaviour of software artefacts. With the upcoming Model Based Engineering there is a need to ensure that the embedded code running on some device is behaving the same way as its Model. In this Thesis we concentrate on UML Activity diagrams used to model C-Components. We transform an Activity with embedded OCL constraints into a rigorous Mathematical program expressed in AMPL and solve it using state-of-the art constraint solvers. The generated tests will comply to control flow based coverage criteria and we can specify to explicitly use boundary values for testing. A special focus is on non linear mixed integer arithmetic constraints.}
